CSS Custom Properties

CSS custom properties — also called CSS variables — let JavaScript reach directly into your stylesheet and change values at runtime. When you flip the switch, one line of JS updates --color-night and the entire scene reacts instantly. Every element using that variable updates simultaneously — no class toggling, no inline style loops.

Move your mouse across the photos after the lights are off to reveal what's hanging in the dark.

// Set a CSS variable at runtime: document.documentElement .style.setProperty( '--color-night', '#080005' ); // Read it back: getComputedStyle(root) .getPropertyValue('--color-night'); // → '#080005' // Every element using that // variable updates immediately.

THE FILING CABINET: localStorage

What it is: localStorage is a key-value store built into every browser. Unlike regular JavaScript variables (which disappear when the page closes), localStorage data persists on the user's device — indefinitely, until the user clears their browser data or you explicitly delete it.

Why it matters: localStorage lets you build experiences that remember users — saved settings, game progress, shopping carts, form drafts, dark/light mode preferences. It holds up to ~5MB per domain and needs no server or login.

The catch: Everything stored is a string. To store objects or arrays, use JSON.stringify() to write and JSON.parse() to read.

// Save case progress:
localStorage.setItem('clues', cluesFound);
localStorage.setItem('suspects', suspectsInterrogated);
// Load it back later:
localStorage.getItem('clues'); // → "3"
// Storing an object:
localStorage.setItem('case',
  JSON.stringify({ clues: 3 }));
// Remove a key:
localStorage.removeItem('clues');
